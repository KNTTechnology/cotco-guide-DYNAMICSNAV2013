คู่มือผู้ใช้ · เมนูการวางแผนการผลิต

วางแผนการผลิตอย่างไรให้ระบบช่วยจัดเวลาให้ถูกต้อง

สรุปขั้นตอนการใช้งานและกฎการจัดตารางเวลาของเมนู การวางแผนการผลิต (Production Plan) แบบเข้าใจง่าย — เพื่อให้ผู้วางแผนรู้ว่าช่องไหนแก้ได้ ช่องไหนระบบคุม และอะไรทำให้บันทึกไม่ได้

เขียว = ผู้ใช้แก้ได้เอง น้ำเงิน = ระบบคำนวณให้ เหลือง = ถูกล็อก / ต้องระวัง แดง = บันทึกไม่ได้จนกว่าจะแก้
ขั้นตอน

ใช้งาน 3 ขั้นตอน

หน้าจอแบ่งเป็น 3 ส่วนตามลำดับ ทำจากบนลงล่าง

1

ตั้งค่าหัวแผน

เลือก เครื่องจักร แล้วกำหนด วันเริ่ม–วันสิ้นสุด ของแผน

จำเป็นต้องกรอกก่อนไปขั้นถัดไป

2

เพิ่มใบสั่งผลิต

เลือกการ์ด RPO จากด้านซ้ายเข้ามาในแผน (ใบที่ถูกจองหรือผลิตครบแล้วจะเพิ่มไม่ได้)

RPO = ใบสั่งผลิตที่ปล่อยแล้ว

3

จัดตารางเวลา

ตั้ง เริ่ม / สิ้นสุด / Setup ของแต่ละแถว ลากสลับลำดับได้ แล้วกด บันทึก

ระบบตรวจและขึ้นป้ายผ่าน/ไม่ผ่านให้ทันที

หัวใจสำคัญ

แถวมี Routing กับ ไม่มี Routing ต่างกันอย่างไร

นี่คือสิ่งที่กำหนดว่าใครเป็นคนคุมเวลาของแถวนั้น — ผู้ใช้ หรือ ระบบ

ระบบคุม

แถว “มี Routing”

สินค้าที่มี มาตรฐานการผลิต (Routing Standard) — ระบบคำนวณเวลาให้จากรอบการผลิตจริง

เวลาเริ่มระบบกำหนดอ่านอย่างเดียว
เวลาสิ้นสุดระบบกำหนดอ่านอย่างเดียว
Setupผู้ใช้ปรับได้แก้ได้

แก้ Setup แล้วระบบจะคำนวณเวลาสิ้นสุดและแถวถัดๆ ไปใหม่ให้เอง

ผู้ใช้คุมเอง

แถว “ไม่มี Routing”

สินค้าที่ยังไม่มีมาตรฐานการผลิต — ผู้วางแผนเป็นเจ้าของเวลาทั้งหมด

เวลาเริ่มผู้ใช้กำหนดแก้ได้
เวลาสิ้นสุดผู้ใช้กำหนดแก้ได้
Setupผู้ใช้กำหนดแก้ได้

ถ้าไม่ตั้งเวลาสิ้นสุดเอง ระบบจะเติมให้อัตโนมัติด้วย บล็อกมาตรฐาน 2 ชั่วโมง (ดูกฎข้อ 3)

กฎ 6 ข้อ

กฎการจัดตารางเวลา

ระบบใช้กฎเหล่านี้จัดเวลาให้อัตโนมัติทุกครั้งที่ผู้ใช้แก้แถวใดแถวหนึ่ง

1

แถวมี Routing — ระบบคุมเวลา

เวลาเริ่ม/สิ้นสุด แก้ไม่ได้ ปรับได้เฉพาะ Setup แล้วระบบคำนวณใหม่ให้

ผู้ใช้เห็นบนจอ
2

แถวไม่มี Routing — ผู้ใช้คุมเวลา

ตั้งเวลาเริ่ม เวลาสิ้นสุด และ Setup ได้เองทั้งหมด

ผู้ใช้เห็นบนจอ
3

บล็อกมาตรฐาน 2 ชั่วโมง

ถ้าไม่มี Routing และไม่ได้ตั้งเวลาเอง: สิ้นสุด = เริ่ม + Setup + 120 นาที โดยข้ามพักเที่ยงและวันหยุดให้

ระบบคำนวณให้
4

เวลาที่ตั้งเองจะไม่ถูกรีเซ็ต

เมื่อตั้งเวลาเริ่ม/สิ้นสุดเอง ระบบจะเก็บค่านั้นไว้ แก้ Setup แล้วเวลาสิ้นสุดจะขยับตามส่วนต่าง ไม่กลับไปบล็อก 2 ชม.

ผู้ใช้เห็นบนจอ
5

ต่อเนื่องกันอัตโนมัติ

แก้เวลาแถวหนึ่ง แถวถัดไปจะเลื่อนตาม — เริ่มของแถวถัดไป = สิ้นสุดของแถวก่อนหน้า

ระบบคำนวณให้
6

ลากสลับลำดับได้

ลากแถวเพื่อจัดลำดับใหม่ ระบบจะคำนวณเวลาให้ใหม่ตามลำดับ (ไม่คงเวลาเดิมของแถว)

ผู้ใช้เห็นบนจอ
เวลา Setup

Setup กับเวลาสิ้นสุด

Setup คือเวลาตั้งเครื่องก่อนเริ่มผลิต — จองไว้หน้างานเสมอ

แก้ Setup บนแถวที่ตั้งเวลาเอง

เวลาสิ้นสุดจะขยับตาม “ส่วนต่าง” ของ Setup เท่านั้น

เวลาสิ้นสุดใหม่ = เวลาสิ้นสุดเดิม + ( Setup ใหม่ − Setup เดิม )
ตัวอย่าง: เวลาสิ้นสุดเดิม15:00
Setup เปลี่ยน 10 → 20 นาที+10
เวลาสิ้นสุดใหม่15:10

กฎที่ต้องผ่านเสมอ

ระบบยึดกฎเดียวกับฝั่งเซิร์ฟเวอร์เพื่อกันข้อมูลผิด

เวลาสิ้นสุด > เวลาเริ่ม
ถ้าลด Setup จนเวลาสิ้นสุดไม่เกินเวลาเริ่ม ระบบจะขึ้นข้อความเตือนสีแดงทันที และปุ่มบันทึกจะกดไม่ได้จนกว่าจะปรับเวลาเริ่ม เวลาสิ้นสุด หรือ Setup ให้ถูกต้อง
สถานะแถว

แถวไหนแก้ได้ แถวไหนถูกล็อก

งานที่เริ่มหรือเสร็จไปแล้ว (มีการรายงานผลผลิตจากหน้างาน) จะถูกล็อกกันแก้ย้อนหลัง

รอผลิต (Pending) · แก้ได้

ยังไม่ถึงคิว — แก้เวลา ลากสลับ และลบได้ตามปกติ

เลยกำหนด (Overdue) · แก้ได้

ถึงเวลาแล้วแต่ยังไม่มีการผลิต — ยังเลื่อน/แก้ได้

กำลังผลิต (Running) · ล็อก

เริ่มผลิตแล้ว — แก้เวลา ลาก หรือลบไม่ได้

ผลิตเสร็จ (Finished) · ล็อก

ผลิตเสร็จแล้ว — ตัวเลขและเวลาที่ทำไปแล้วแก้ไม่ได้

การล็อกดูจากค่าที่บันทึกล่าสุด — ที่แก้ค้างไว้แต่ยังไม่บันทึกจะไม่ทำให้แถวถูกล็อก

บันทึกไม่ได้

สิ่งที่ทำให้ปุ่มบันทึกกดไม่ได้

แก้ให้ครบทุกข้อ แล้วป้ายจะเปลี่ยนเป็น “ผ่านทั้งหมด” และบันทึกได้

เวลาสิ้นสุดไม่เกินเวลาเริ่ม — แต่ละแถวเวลาสิ้นสุดต้องมากกว่าเวลาเริ่ม
แถวใบสั่งผลิตไม่มีรายการสินค้า — ต้องมีสินค้าอย่างน้อย 1 รายการ พร้อมเวลาเริ่ม/สิ้นสุด
จำนวน / น้ำหนัก ไม่ถูกต้อง — จำนวนต้องมากกว่า 0 และน้ำหนักอย่างน้อย 0.001 กก.
วันที่ของแถวอยู่นอกช่วงแผน — ต้องอยู่ในช่วงวันเริ่ม–สิ้นสุดของแผน
เวลาชนกันบนเครื่องเดียวกัน — ห้ามสองแถวใช้เครื่องเดียวกันในเวลาที่คาบเกี่ยว (รวมถึงแผนอื่นบนเครื่องนั้น)
เริ่มแผนในอดีต หรือเริ่มวันอาทิตย์ — (เฉพาะแผนใหม่) ห้ามวันเริ่มเป็นอดีตหรือวันอาทิตย์
