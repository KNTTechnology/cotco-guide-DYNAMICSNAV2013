คู่มือ 02 · Production / Consumption / Output

บันทึกการผลิตหน้างานแบบเรียลไทม์

คู่มือสำหรับผู้ปฏิบัติงานหน้าเครื่อง — เริ่ม/หยุด/พักงาน, บันทึกวัตถุดิบที่ใช้ (Consumption), บันทึกผลผลิตและออก Lot (Output) ทุกอย่างถูกบันทึกเข้าระบบทันทีและรอส่งเข้า Navision 2013

เขียว = ผู้ใช้บันทึกได้ น้ำเงิน = ระบบ/NAV กำหนด เหลือง = ล็อก / ต้องระวัง แดง = บันทึกไม่ได้จนกว่าจะแก้
ภาพรวม

ลำดับงานหน้าเครื่อง

งานหนึ่งใบไหลผ่าน 4 ขั้น — เริ่มจากแผนที่วางไว้

1

Production

เริ่ม–พัก–หยุด งานผลิตบนเครื่อง จับเวลาจริง

2

Consumption

บันทึกวัตถุดิบที่ใช้ไป (เบิก WIP)

3

Output

บันทึกผลผลิต ออก Lot พิมพ์สติกเกอร์

4

QC

ตรวจสอบคุณภาพ (แนบเทมเพลต)

A · Production

บอร์ดผลิตหน้าเครื่อง

เลือกเครื่องจักรและวันที่ แล้วดูคิวงานที่มาจากแผนการผลิต

คิวงานอัปเดตเองทุก ~7 วินาที

บอร์ดของวันนี้จะรีเฟรชอัตโนมัติ แท็บเล็ตอีกเครื่องเห็นการเปลี่ยนแปลงโดยไม่ต้องกดรีเฟรช · เลือกวันย้อนหลังเพื่อดูงานที่ตกค้างได้ (แต่หน้าย้อนหลังจะดูอย่างเดียว แก้ไม่ได้) · เลือกวันในอนาคตไม่ได้

·

สถานะงานบนบอร์ด

กำลังผลิต (Running) พักงาน (Paused) ผลิตเสร็จ (Finished) รอเริ่ม (Waiting) อนุมัติแล้ว (Approved) ถูกปฏิเสธ (Rejected) ค้างการผลิต (Overdue)
·

ปุ่มควบคุมงาน

เริ่มงานได้เฉพาะใบที่ “อนุมัติแล้ว” และเป็นบอร์ดวันนี้

เริ่ม / ดำเนินการ

Start · Stop

Start เริ่มจับเวลาการผลิต · Stop จบงาน → สถานะเป็น “ผลิตเสร็จ”

พัก / ดำเนินต่อ

Pause · Resume

Pause ต้องเลือกเหตุผล (จากรายการ OEE ของเครื่อง) เพิ่มหมายเหตุได้ · Resume กลับมาผลิตต่อ

อนุมัติงาน

Approve · Reject

ผู้มีสิทธิ์อนุมัติเท่านั้น · Reject ต้องใส่เหตุผล · ต้องอนุมัติก่อนถึงจะกด Start ได้

สิ่งที่ล็อก / ทำไม่ได้

บอร์ดวันย้อนหลัง = ดูอย่างเดียว (ปุ่มควบคุมซ่อน) · งานที่กำลังผลิตจะลากสลับลำดับไม่ได้ · ลากสลับได้เฉพาะภายในใบเดียวกันบนบอร์ดวันนี้ · เครื่องแบบ Parallel เวอร์ชันนี้ปุ่มควบคุมเป็นแบบอ่านอย่างเดียว

แถบ Main / Capacity / Consumption / Output / QC ในแผงรายละเอียด ให้กดข้ามไปทำแต่ละขั้นของงานนั้นได้ · Capacity แสดงเวลาเดินเครื่อง เวลาสูญเสีย และประวัติการพัก

B · Consumption

บันทึกวัตถุดิบที่ใช้

บันทึกการเบิกวัตถุดิบของใบสั่งผลิต — คลัง WIP

1

เลือกงาน

เลือกเครื่อง → ใบสั่งผลิต (RPO) → รายการสินค้า

2

เพิ่มวัตถุดิบ

สแกน QR / ค้นหา / เลือกจากสูตร (BOM)

3

ใส่จำนวน

กรอกน้ำหนัก + จำนวนที่ใช้ (มีปุ่มคำนวณช่วย) เพิ่ม Scrap ได้

4

บันทึก

ตั้งวันที่ + คลัง = WIP → กดบันทึก

·

เงื่อนไขที่ต้องผ่าน

คลังต้องเป็น WIP — คลังอื่นจะถูกปฏิเสธ
น้ำหนักและจำนวนต้องมากกว่า 0
วัตถุดิบต้องอยู่ในสูตร (BOM) ของใบสั่งนั้น — ใบสั่งต้องเป็นสถานะ Released, ล็อตต้องมีในสต็อก (ระบบตรวจกับ Navision ตอนสแกน/ค้นหา)
ใช้ได้ไม่เกิน 105% ของที่สูตรกำหนด — เกินกว่านั้นจะขึ้น error

บันทึกแล้วแก้ไม่ได้ → ใช้การกลับรายการ

รายการที่บันทึกแล้วจะล็อก เพื่อรักษาความถูกต้องของข้อมูล ถ้าใส่ผิดต้องกลับรายการ (Reverse) — ระบบจะลงรายการติดลบหักล้าง ไม่ใช่การลบทิ้ง (ฟังก์ชันนี้ถูกซ่อนไว้ในช่วง UAT)

C · Output

บันทึกผลผลิต และออก Lot

บันทึกของที่ผลิตเสร็จ พร้อมพิมพ์สติกเกอร์ประจำล็อต

1

เลือกงาน + สร้าง Lot

เลือกเครื่อง → RPO → สินค้า แล้วกด Generate Lot ระบบตั้งเลข Lot + เติมจำนวนคงเหลือให้

2

กรอกข้อมูลล็อต

น้ำหนัก (กก.) · จำนวน · คุณลักษณะสินค้า (ผิว/ขอบ/เกรด/สเปก ฯลฯ) ให้ครบ

3

บันทึก + พิมพ์

กด Save Output แล้วดู/พิมพ์สติกเกอร์ของล็อตได้

·

เงื่อนไขที่ต้องผ่าน

ผลผลิตรวมห้ามเกินจำนวนที่วางแผนไว้ — ไม่มีเผื่อ (เกิน = OverProduction)
น้ำหนักรวมเกินเป้า → ช่องเป็นสีแดง — และปุ่มบันทึกจะกดไม่ได้
ต้องกรอกทุกช่องของล็อตให้ครบ — เลข Lot, น้ำหนัก > 0, จำนวน > 0 และคุณลักษณะครบ
เลข Lot ต้องไม่ซ้ำ — และใบสั่งต้องเป็นสถานะ Released

ล็อตที่บันทึกแล้วจะล็อก — แก้ไขได้เฉพาะตอนที่ยังเป็นสถานะ รอส่ง (Pending) เท่านั้น · ล็อตที่ยังไม่บันทึกลบได้ ล็อตที่บันทึกแล้วลบไม่ได้

D · รายงาน

Capacity · OEE · รายงานประจำวัน

สรุปประสิทธิภาพการผลิตของเครื่องในแต่ละวัน

Availability

%

เวลาเดินเครื่องจริง

Performance

%

ความเร็วเทียบมาตรฐาน

Quality

%

ของดีต่อของทั้งหมด

OEE รวม

%

ประสิทธิผลโดยรวม

เลือก เครื่อง + วันที่ แล้วดูรายงานประจำวัน (พิมพ์ได้) — มี 4 ส่วน: สรุปการผลิต, การใช้วัตถุดิบ, ผลผลิต และเวลาสูญเสีย

E · Navision

ข้อมูลส่งเข้า Navision 2013 อย่างไร

บันทึกที่หน้างานทั้งหมดถูกเก็บทันที แล้วรอส่งเข้า NAV

ขาออก → NAV

สิ่งที่เราบันทึก

การเริ่ม/พัก/หยุด, การใช้วัตถุดิบ และผลผลิต ถูกบันทึกเข้าฐานข้อมูลของ Cotco ทันที ด้วยสถานะ “รอส่ง (Pending)”

จากนั้นผู้ใช้กด Post เพื่อดึงรายการเหล่านี้เข้า NAV 2013

ขาเข้า ← NAV

ข้อมูลอ้างอิง

เครื่องจักร, ใบสั่งผลิต, สูตรวัตถุดิบ (BOM), สต็อกล็อต และเหตุผลการพัก อ่านสดจาก Navision

หน้างานจึงตรวจกับข้อมูล NAV ปัจจุบันเสมอ (ใบสั่งต้อง Released, วัตถุดิบต้องอยู่ใน BOM, ต้องมีสต็อก)

แก้ไข = กลับรายการ ไม่ใช่การลบ

รายการที่ลงผิดจะถูกแก้ด้วยการกลับรายการ (Reverse) ที่ลงตัวเลขติดลบหักล้าง เพื่อให้มีร่องรอยตรวจสอบย้อนหลังได้ และซิงก์เข้า NAV อย่างถูกต้อง
